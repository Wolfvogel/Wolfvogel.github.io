Basic JavaScript

Calculations

For adding, subtracting, multiplying and dividing, all the following examples would declare a variable named value, then set it to 15;

var value = 5 + 10;
var value = 20 - 5;
var value = 5 * 3;
var value = 30 / 2;

When a variable has a set value, this can be incremented or decremented by 1, in both of these cases resulting in variable value set to 15;

var value = 14;
value++;

var value = 16;
value--;

It can also have a number added to, subtracted from, divided by or multiplied with it, and then have its value set to the result. There are operators which do this in one step, resulting in the following all resulting in variable value set to 15;

var value = 20;
value -= 5;

var value = 11;
value += 4;

var value = 45;
value /= 3;

value = 7.5;
value *= 2;

The remaining number after dividing two numbers can be found with the remainder operator. It can be used to check for even or odd numbers, by getting the remainder of division by 2. It does not work properly with negative numbers. In this case, variable value would be set to 6;

var value = 13 % 7;

Strings

A series of characters enclosed in (single or double) quotes is called a string. A string is immutable, meaning that once created the individual characters can't just be changed. The only way to change a string, is to assign it with a new string.

var string = "Hello";

When you need to use a single or double quote inside your string, it can be escaped using a \ so JavaScript knows it's not the end of the string. It's also possible to use single quotes within a string that starts and ends with double quotes, and vice versa.

var string = 'I\'m a string';
var string = "I'm a string";

Other things can be represented in a string by using the backslash escape, like;

Backslash: \\
New line: \n
Carriage return: \r (Resets to the beginning of a line of text)
Tab: \t
Word boundary: \b (?)
Form feed: \f (Advance downwards to the next 'page', used as section separators or page breaks)

Strings can be concatenated with +, or concatenated onto the end of an existing string with +=. Concatenating strings does not add spaces between the strings, so make sure to add these yourself where necessary. Strings can also be concatenated with variables.

var doubleString = "This is a " + "concatenated string.";

var doubleString = "This is also a ";
doubleString += "concatenated string.";

var holiday = "Halloween";
var longString = "My favourite holiday is " + holiday + ", what is yours?";

The length of a string value can be found with .length;

var catNameLength = 0;
var catName = "Calida";
catNameLength = catName.length;

It is also possible to find a specific character in a string. For this, remember that computers start counting at 0; the first character in a string is the zeroeth character for your script.

var firstLetter = "";
var catName = "Calida";
firstLetter = catName[0];

var thirdLetter = "";
var catName = "Calida";
thirdLetter = catName[2];

var catName = "Calida";
var lastLetter = catName[catName.length - 1];

var catName = "Calida";
var secondToLastLetter = catName[catName.length -2];

Arrays

It's possible to store multiple pieces of information in a variable, using an array. An array declaration starts and ends with square brackets, and have a comma between each entry. Arrays can also hold other arrays, called a multi-dimensional array;

var pets = ["Cat", 2], "Dog", "Rat", 3]];

The data inside an array can be accessed using indexes, which are written in the same bracket notation. Like strings, they start counting at 0.

var shoppingList = ["Milk", "Bread", "Eggs"];
shoppingList[1]; //equals "Bread"

var multiArray = [[1, 2, 3], [[4, 5], 6], 7, 8, 9];
multiArray[3]; //equals 8
multiArray[0]; //equals [1, 2, 3]
multiArray[1][0][1]; //equals 5

Unlike strings, arrays are mutable and can be changed;

var shoppingList = ["Milk", "Bread", "Eggs"];
shoppingList[2] = "Jam"; //equals ["Milk", "Bread", "Jam"]

Data can be added to the end of an array via the push() function;

var array = [1, 2, 3];
array.push(4); //array equals [1, 2, 3, 4]

The pop() function can be used to remove the last value of an array, and can be used to store that value in a variable.

var array = [1, 2, 3];
var lastValue = array.pop(); //array equals [1, 2]; lastValue equals 3

The shift() function works similar to pop(), except that it removes the first value instead of the last one.

var array = [1, 2, 3];
var firstValue = array.shift(); //array equals [2, 3]; firstValue equals 1

The unshift() function can then be used to add a value to the beginning of an array;

var array = [1, 2, 3];
array.unshift(0); //equals [0, 1, 2, 3]

Functions

Functions are reusable parts of JavaScript code that can be invoked by using its name, at which point all of the code between the curled braces will be executed.

function greetings() {
  console.log("Hello World");
}
greetings(); //Outputs "Hello World"

When defining a function, one or more parameters can be defined along with it. Parameters act as placeholders, for the values that will be input into the function when it is called.

function subtraction(a, b) {
  console.log(a - b);
}
subtraction(15, 2); //Outputs 13
subtraction(30, 10); //Outputs 20

Variables declared within a function, and the function parameters, are created with a local scope - meaning they are only visible within that function. It is possible to create both local and global variables with the same name, in which case the local variable always takes precedence.

var animal = "Elephant";
function favoriteAnimal() {
  var animal = "Giraffe";
  return animal;
} //Outputs "Giraffe"

Where values can be passed into a function with arguments, or parameters, a return statement can be used to send a value back out of a function.

var timesFive(number) {
  return number * 5;
}
var result = timesFive(2); //Outputs 10
var result = timesFive(6); //Outputs 30

When a function does not have a return statement, upon being called it processes the inner code, but the returned value is undefined.

var sum = 0;
function addSum(num) {
sum = sum + num;
}
addSum(5)//Sum is modified, but returned value is undefined

Comparisons

There are many comparison operators in JavaScript, which all return a boolean true or false value.

  • Equality operator (==) Compares two values and returns true if they're equivalent
      (3 == '3' // true)

  • Strict equality operator (===) Compares two values and returns true if they're exactly equal, without attempting to convert both values to a common type
      (3 === 3 // true)

  • Inequality operator (!=) Compares two values and returns true if they're not equivalent.
      (3 != 4) // true
      (3 != '3' // false)

  • Strict inequality operator (!==) Compares two values and returns true if they're not exactly equal, without attempting to convert both values to a common type
      (3 !== '3' // true)
      (3 !== 3) // false

  • Greater than operator (>) Compares two numbers, and returns true if the number on the left is greater than the number on the right. It will also convert data types when comparing.
      (3 > '2' // true)
      (3 > 3 // false)

  • Greater than or equal to operator (>=) Compares two numbers, and returns true if the number on the left is greater than or equal to the number on the right. It will also convert data types when comparing.
      (3 >= '3' // true)
      (1 >= 3 // false)

  • Less than operator (<) Compares two numbers, and returns true if the number to the left is less than the number on the right. It will also convert data types when comparing.
      (3 < '4' // true)
      (3 < 2 // false)

  • Less than or equal to operator (<=) Compares two numbers, and return true if the number on the left is less than or equal to the number on the right. It will also convert data types when comparing.

If and Else If

The if statement is used to execute code when certain conditions, defined in parentheses, are met. These are called boolean conditions, and can only be true or false. When the boolean condition comes to true, the code in the curled braces is executed and the result is returned, bypassing the rest of the function. When the boolean condition comes to false, the code in the curled braces is not executed, and the function continues.

function test (condition) {
  if (condition == 5) {
    return "Equal";
  }
  return "Not equal";
}
//Returns Equal when boolean condition is true (5 == 5 / '5' == 5)
//Returns Not equal when boolean condition is false (1 == 5)

With the logical and operator (&&) it is possible to test multiple things at the same time. It returns true only if the both of the operanders next to it are true.

function 3to6 {
  if (num > 3 && num < 6) {
    return "Yes";
  }
  return "No";
}
// Returns "Yes" if num is greater than 3 but less than 6, and "No" if it is 3 or less, or 6 and greater.

The logical or operator returns true if either of the operanders next to it is true.

function 3or9 {
  if num === 3 || num === 9 {
    return "Yes";
  }
  return "No";
}
// Returns "Yes" if num is exactly equal to 3 or 9, and "No" if it is anything else.

When a condition for an if statement is true, it's code is executred. When the condition is false, normally nothing happens. With an else statement, an alternate block of code can be executed. If multiple conditions need to be addressed, if statements can be chained used else if statements. The function if executed from top to bottom, so you need to take care which statements come first.

function threeOrNot(num) {
  if (num > 3) {
    return "Bigger than 3";
  } else if (num == 3) {
    return "3";
  } else {
    return "Less than 3";
  }
}

To return a boolean true or false value, instead of using an if/else statement, it's often shorter to use a strict equality operator.

function isEqual(a,b) {
  if (a === b) {
    return true;
  } else {
    return false;
  }
}
// Can also be written as;

function isEqual(a,b) {
  return a === b;
}

Switch Statements

A switch statement can test a value against many predefined possible cases, using strict comparison (===). It executes statements from the first matched case value until it encounters a break. Cases can be chained, when several cases would produce the same return (in the case below, for values 3 and 4). A default statement can be added, which will be executed if no other cases match the input value.

function(value) {
  var answer="";
  switch (val) {
   case 1: return "one"; break;
   case 2: return 2; break;
   case 3: case 4: return "three or four"; break;
   case "unknown": return "that is not a number"; break;
   default: return "other"; break;
  }
  return answer;
}

Objects

Objects are similar to arrays, except data is accessed through properties instead of by using indexes. They are useful for storing data in a structured way. The properties ("name", "colours", etc) can be stored as strings, but numbers can also be used as properties, and for single-word string properties the quotes can even be omitted.

var unicorn ={
  name: "Moonshine",
  "main colours": ["Blue", "White", "Lilac", "Silver"],
  legs: 4,
  horn: 1
};

The properties of an objects can be accessed via dot notation (.) and bracket notation ([]). Dot notation is used when you know the name of the property you're trying to access, and it's a number or a one-word string. For properties with a space in the name, bracket notation is used. This can also be used on object properties without spaces.

var unicornName = unicorn.name; // "Moonshine"
var unicornColours = unicorn["main colours"]; // ["Blue", "White", "Lilac", "Silver"]

It's also possible to access an object property by using variables. This can be useful when accessing a lookup table

var plushies = {
  Charlie: "Red panda",
  Mini: "Spider",
  Hinnik: "Horse"
};
var myPlushie = "Charlie";
var whatSpecies = plushies[myPlushie];
console.log(whatSpecies); // "Red panda"

The properties of an object can be changed, just like a variable, using either dot or bracket notation. Additionally, it is possible to add new properties to an object, or to remove properties.

plushies.Mini = "Large Spider";
// Changes the above object's Mini property to "Large Spider"

plushies["Om"] = "Cat";
// Adds the Om property to the plushies object, and sets it to "Cat"

delete plushies.Hinnik;
// Deletes the Hinnik property from the plushies object

It's possible to check if an object has a specific property;

plushies.hasOwnProperty("Charlie";) // true plushies.hasOwnProperty("Bella"); // false
